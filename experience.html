Information Systems 2020-2021 Lab Project
Preperation
Firstly, we should install QGIS and the virtual machine and download the data needed for starting the project. When installing the virtual machine, we should be aware that we do not change its path folder or update our windows because it causes some running software problems. Otherwise, we faced an error message when we want to start our software. Data includes two parts, Global Human Settlement - Population Grid and WorldPop - Population Counts. The first part contains a spatial raster dataset that depicts population distribution and density, expressed as the number of people per cell. Another one represents the estimated total number of people per grid-cell set on the raster file dataset. In this project, we intend to comparison our data and validate our results in five steps. Each group developed a website in the last step that depicts our outputs, project information, etc. In the following, we will deal with the project stages.
Step1
We are going to clip and resample our data, which was downloaded in advance. After uploading them on QGIS and loading our group's sub-area via the GIS-lab areas file (gpkg file). We executed the clip and critical to maps comparison; it is called align. Besides, we simplified further map computations, using integer because of avoiding to use float values in this step.
Moreover, we must be adjusted null values for both new files (GHS-clip and worldpop-clip). The null value of the worldpop-clip file is equal to zero because of seas, oceans, and inland water. On the other hand, another value for GHS-clip is -200 because of seas and oceans. So, we sat null values via raster calculator; then we should check metadata in the processing part with the r.null tool, which updated correctly (attention: need to open QGIS with Grass in this part, otherwise we faced an error message).
Step2
We computed differences between two files and the statistical discrepancy of two maps. We assist r.univar in computing differences between our maps, so we computed a correlation matrix for each sub-area tile during the process. Additionally, we add the new attribute field to our tiles vector layer, which indicates that the oﬀ-diagonal value of the correlation matrix is stored in our project(attention: we can get various outputs during this part of the process). We use r.covar to compute the correlation matrix for each tile because we have to Clip the raster maps using one geometry per time of the tile vector layer as a mask. But we should be noted that adding a correlation factor is manual. Our results in this step are TXT, or CSV file includes differences of maps and vector layer(in gpkg format) with the correlation values in the attribute table computed for each tile.
Step3
We are going to reclassify both files in this part to obtain thematic maps. For this purpose, we check the pixel values; if they are positive, we will set them to mark 'built,' and on the other hand, if they equal to zero, set to mark 'not built.' We go to the raster calculator again and input the code in this part. After that, setting unique value for new files that reclassified them in advance via symbology part of layer properties. Furthermore, we should distinguish differences between reclassiﬁed GHS-POP and WorldPop and adding one to them. We consider three variant conditions: firstly, one means that both files are built; secondly, zero means that GHS is not populated, but WordPop populated; and two is inverse. We input the code, which indicates to last numbers explain before, in the raster calculator. Finally, we should integer our raster file and setting unique values style.
Step4
Briefly, we will check out the accuracy of thematic maps via the AcATaMa plugin. So, we create random samples and using diﬀerence raster for stratiﬁcation. checking 600 pieces includes
- 66 of points where GHS not populated, but WordPop populated
- 68 of points where GHS and WordPop either both populated or unpopulated
- 66 of points where WordPop not populated, but GHS populated.

After that, we classify samples by interpreting pictures via Bing Satellite and Google Satellite. Hence, we compute the accuracy of a map and get the result value of the second map in the sample points' location. And in the last step, we calculate the accuracy of the second mao via a Python script. We start to process validation with the AcATaMa plugin that we accessed in the plugin part. Because of the bug in this step, we have to choose a difference raster file instead of thematic maps to create a sampling scheme, and also, the plugin requires that type of raster data must be byte or integer with non zero positive values and unique value or singleband pseudocolor style. Moreover, we should create a sampling scheme in another step via AcATaMa. We enter the number of samples in this part. Still, we must choose the minimum distance between samples is 0.08 because it causes samples to have ten pixels away from each other, and after that, we can click on Generate the sampling to generate sampling points. Again back to the thematic part and choosing raster, which we will validate. Before classification, we need high-resolution imagery, so we use Google satellite and Bing satellite on the QuickMapServices part plugin. We should set the classification name(we divided two classes: urban and not urban) and color for each type and class value in the thematic raster in another step. We can render two VHR data from Bing Satellite and Google Satellite (two satellites help us if we have an unobvious picture); additionally, we should adjust the window size to 0.004 deg. With this setting display the measure will be around pixel size, which is important because class estimated on pixel-level that we sat in advance. We survey our VHR images and check them for classification; when they are completed, we can save sampling and classification and load them for continuing other parts. We select ﬁle of sampling points with classiﬁcation, and we click on Open the accuracy assessment results to get accuracy metrics estimated by the AcATaMa plugin. The limitation of the AcATaMa plugin is that we can only estimate the accuracy of the deﬁned thematic raster. Still, we cannot reuse samples to evaluate the accuracy of another map. For this reason, we will use Python scripts to overcome this issue. It allows the computation of the error matrix and accuracy indexes a map starting from the vector of points, which contains one attribute ﬁeld with classiﬁed validation samples. We can see the user's accuracy in the first part of the error matrix, and we also get the error matrix file via Python script that shows at the bottom.

Firstly，we just downloaded the template website from the website because of time consuming，we also learned some brief knowledge from the template and it’s very convenient for us.For example, I learned how to insert phases and images, also, Including some ways to adjust the space deviation.

(2)openlayer+geoserver
In this part we have to Combine the operations between openlayer and geoserver.At the beginning, we had to upload our lab results to geoserver With their own styles, And we could check if the data has some troubles by preview windows, If we can see as we thought in QGIS, then the this part is fine.
For the second part we have to write the code of JavaScript for building webgis, It included that adding controls, Three base maps And all the lab results from geoserver. At this step, we might meet many of the problems and we have to find the troubles and fix them, At this time, the developer tool Is a good helper that can help us via console and networks control.
